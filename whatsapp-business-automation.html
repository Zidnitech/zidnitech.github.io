01 // Overview

This reference documents the technical layer of a WhatsApp business automation system. The commercial strategy behind each component — why to build it, what ROI to expect, which markets it serves — lives in the Arabic-language guides linked throughout. This page covers the how.

The system is built around four primitives:

  • Webhooks — inbound event stream from Meta and your business systems
  • Interactive Messages — Reply Buttons and List Messages that drive structured user choices
  • Broadcast Pipelines — segmented outbound campaigns at scale
  • Automation Connectors — Zapier / Make flows that bridge WhatsApp with your CRM, store, and calendar

02 // Webhooks & Integration

Webhooks are the nervous system of any WhatsApp automation. Meta fires a POST to your endpoint for every inbound message, delivery receipt, and button click. Your business systems fire their own events (order placed, appointment booked, payment received) that trigger outbound messages.

The Arabic guide on ما هو Webhook والـ Integration وكيف يربطان واتساب بأنظمتك covers the business logic and ROI behind this architecture in full detail — treating webhooks not as a technical concept but as the foundation of a customer communication infrastructure.

Incoming Webhook — Signature Verification

JS src/webhooks/verify.js 
const crypto = require('crypto');

/**
 * Verify Meta HMAC-SHA256 signature
 * Required for every inbound webhook — reject anything that fails
 */
function verifyMetaSignature(rawBody, sigHeader) {
  const expected = 'sha256=' + crypto
    .createHmac('sha256', process.env.META_APP_SECRET)
    .update(rawBody)
    .digest('hex');

  return crypto.timingSafeEqual(
    Buffer.from(expected),
    Buffer.from(sigHeader)
  );
}

/**
 * Express middleware — attach rawBody for signature check
 */
const rawBodyParser = express.json({
  verify: (req, res, buf) => { req.rawBody = buf; }
});

/**
 * Main webhook receiver
 */
router.post('/webhook', rawBodyParser, async (req, res) => {
  const sig = req.headers['x-hub-signature-256'];

  if (!verifyMetaSignature(req.rawBody, sig)) {
    return res.status(401).end();
  }

  res.status(200).end(); // Acknowledge immediately
  await processEvents(req.body);
});

Event Classifier — Route Inbound Messages

JS src/webhooks/classifier.js 
async function processEvents(payload) {
  const entry = payload.entry?.[0];
  const changes = entry?.changes?.[0]?.value;
  if (!changes) return;

  // Delivery / read receipts
  if (changes.statuses) {
    await handleStatuses(changes.statuses);
    return;
  }

  const msg = changes.messages?.[0];
  if (!msg) return;

  const handlers = {
    'text'        : handleText,
    'interactive' : handleInteractive,  // button / list reply
    'image'       : handleMedia,
    'document'    : handleMedia,
    'location'    : handleLocation,
    'order'       : handleOrder,        // catalog order
  };

  const handler = handlers[msg.type] || handleUnknown;
  await handler(msg, changes.contacts?.[0]);
}

/**
 * Handle list selection
 */
async function handleInteractive(msg) {
  const id = msg.interactive.list_reply?.id
           || msg.interactive.button_reply?.id;

  if (id?.startsWith('book_')) {
    const service = id.replace('book_', '');
    await sendTimeSlots(msg.from, service);
  } else if (id?.startsWith('confirm_')) {
    await confirmOrder(id.split('_')[1]);
  }
}
Limits: Reply Buttons = max 3 options, title max 20 chars. List Messages = max 10 sections × 10 rows each, title max 24 chars. Use List when choices > 3.

04 // Broadcast Pipeline

Broadcast is not bulk SMS. It's a segmented, template-gated, consent-based pipeline. Architectural mistakes here result in account suspension — not just poor delivery rates.

The commercial strategy — segmentation logic, Ramadan timing, win-back sequences, and conversion benchmarks for the Saudi market — is covered in depth in the Arabic guide: دليل WhatsApp Broadcast Marketing الكامل.

Broadcast Worker — Rate-Limited Dispatch

JS src/workers/broadcast.worker.js 
const RATE_LIMIT  = 80;   // messages per second (Tier-2 default)
const COOLDOWN_MS = 7 * 24 * 60 * 60 * 1000; // 7-day suppression

/**
 * Process a broadcast campaign batch
 * Each contact: check consent + cooldown → send → log
 */
async function processBroadcastBatch(job) {
  const { contacts, templateName, lang, campaignId } = job.data;
  const results = { sent: 0, skipped: 0, failed: 0 };
  const delay = 1000 / RATE_LIMIT;

  for (const contact of contacts) {
    // 1. Consent check
    if (!contact.marketingOptIn) { results.skipped++; continue; }

    // 2. Cooldown check (Redis)
    const lastSent = await redis.get(`promo:${contact.phone}`);
    if (lastSent && Date.now() - lastSent < COOLDOWN_MS) {
      results.skipped++;
      continue;
    }

    try {
      // 3. Send
      await sendTemplate(contact.phone, templateName, lang, buildComponents(contact));

      // 4. Log cooldown
      await redis.set(`promo:${contact.phone}`, Date.now(), 'EX', 604800);
      results.sent++;
    } catch (e) {
      results.failed++;
      logger.error('Broadcast send failed', { phone: contact.phone, error: e.message });
    }

    // 5. Rate limit
    await sleep(delay);
  }

  return results;
}
Tier limits: New numbers start at Tier-1 (1k messages/day). Tier-2 (10k/day) after quality rating stays Green for 7 days at Tier-1 volume. Never send to opted-out contacts — a single spam report spike can drop you to Tier-0 (blocked).

/**
 * POST /trigger/:flowName
 * Zapier calls this endpoint directly to fire any automation flow.
 * Useful for external tools that can make HTTP requests but don't
 * have a native WhatsApp integration.
 */
router.post('/trigger/:flowName', authMiddleware, async (req, res) => {
  const { flowName } = req.params;
  const data = req.body;

  const allowedFlows = [
    'cart_abandoned',
    'appointment_reminder',
    'payment_failed',
    'lead_followup',
    'review_request',
  ];

  if (!allowedFlows.includes(flowName)) {
    return res.status(400).json({ error: 'Unknown flow' });
  }

  const job = await queue.add(flowName, data, {
    jobId: `${flowName}-${data.id || Date.now()}`
  });

  res.json({ queued: true, jobId: job.id });
});

07 // Common Patterns & Pitfalls

Pattern: Deduplication with Job IDs

Store webhooks sometimes fire twice on retry. BullMQ's jobId prevents duplicate messages — if the job ID already exists in the queue, the duplicate is silently dropped.

JS
// Always set jobId to a deterministic, event-unique value
await queue.add('cod_confirmation', data, {
  jobId: `cod-${data.orderId}`   // idempotent
});

Pattern: Phone Normalization (Saudi)

JS
function toE164Saudi(raw) {
  const d = raw.replace(/\D/g, '');
  if (d.startsWith('966'))  return d;
  if (d.startsWith('05'))   return '966' + d.slice(1);
  if (d.startsWith('5'))    return '966' + d;
  return null; // unrecognized — do not send
}
